# digital-ema

高校文化祭の「神社」展示のために開発された、願い事をデジタル絵馬として奉納・共有できるWebアプリケーション。

## 主な機能

- **絵馬の奉納**: ニックネームと想いを入力すると、大型モニターに絵馬として表示される。
- **リアルタイム通知**: 新しい想いが奉納されると、画面上部に「○○さんの想いが奉納されました」と表示される。
- **当日限定表示**: メイン画面には「今日奉納された想い」のみがランダムで15枚表示される（30秒ごとに入れ替え）。
- **過去アーカイブ**: 別画面で全日程の想いを時系列で閲覧可能（45秒ごとに自動更新）。
- **モバイル対応**: スマートフォンでも見やすいレスポンシブデザイン。

## セットアップ方法

1. Googleフォームを作成し、以下を設定する。
   - 「ニックネーム」（記述式）
   - 「想い」（段落）
2. フォームの回答をGoogleスプレッドシートに連携する。
3. スプレッドシートの「拡張機能」->「Apps Script」でGASプロジェクトを開く。
4. 本リポジトリの `コード.gs`、`index.html`、`archive.html` をコピーする。
5. **`コード.gs` の `SPREADSHEET_ID` を自身のスプレッドシートIDに書き換える。**
6. 「デプロイ」->「新しいデプロイ」でWebアプリとして公開する。
   - 種類: 「ウェブアプリ」
   - アクセスできるユーザー: 「全員」
7. 発行されたURLにアクセスする。

## 絵馬画像の差し替え

`index.html` および `archive.html` 内の以下を変更することで、絵馬の背景画像を差し替え可能。

```css
.ema-card {
  background-image: url('https://example.com/your-ema-card.jpg');
}
```

## 主な仕様

| 項目 | メイン画面 | アーカイブ画面 |
|:---|:---|:---|
| 表示データ | 当日の想いのみ | 全日程の想い |
| 表示枚数 | 最大15枚（5列×3行） | 全件（スクロール） |
| 並び順 | ランダム | 時系列（古い順） |
| 更新間隔 | 30秒ごと | 45秒ごと |
| 通知 | あり（20秒ごと） | なし |

## 当日の運用フロー

来場者とシステムの関わりは以下の通り。

| ステップ | 来場者の動き | システムの動き／部員の対応 |
|:---|:---|:---|
| **奉納** | 備え付けのChromebooまたはQRコードのリンクからニックネームと想いを入力し、絵馬を「奉納」する。 | データがスプレッドシートに保存される。 |
| **通知** | 大型モニターを見上げる。 | 画面上部に「○○さんの想いが奉納されました」と表示される。 |
| **閲覧** | 自分の想いが画面に現れるのを待つ。 | 30秒ごとに、当日の想いからランダムに15枚が表示される。 |
| **アーカイブ** | サブモニターで過去の想いを自由にスクロールする。 | 45秒ごとに全日程の想いが時系列で更新・表示される。|

## カスタマイズポイント

| 変更したい内容 | 変更箇所 |
|:---|:---|
| 表示枚数 | `DISPLAY_COUNT`（index.html） |
| 更新間隔 | `setInterval(updateEmaGrid, 30000)` の数値 |
| 通知の最大人数 | `getRecentNicknames()` 内の `5` |
| 文字数制限 | `maxLen`（デスクトップ5文字／モバイル3文字） |

## 開発の変遷

- **Gen.1**: GASウェブアプリとして誕生。動的表示と自動更新の基盤を構築。
- **Gen.2**: 「ニックネーム」に対応し、「○○さんの想い」として表示。
- **Gen.3**: 運用最適化。GASの制限を考慮した更新間隔の調整。
- **Gen.4**: アーカイブ機能と同時表示数の試験的導入を開始。
- **Gen.5**: アーカイブの正式実装、同時表示20枚化、新着通知機能の追加。
- **Gen.6（最新）**: 15枚・5列表示への最適化、30秒更新、モバイル対応、文字数制限の導入。
